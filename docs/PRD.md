# 人脸识别考勤系统 — PRD (Product Requirements Document)

## 1. 产品概述

### 1.1 一句话定位
基于人脸识别的课堂考勤管理系统——摄像头拍照即可完成签到，自动生成出勤统计。

### 1.2 目标用户
- **管理员（admin）**：系统唯一角色，负责学生信息管理、课程设置、查看考勤数据。
- 系统 **没有** 学生端或教师端，所有操作由管理员完成。

### 1.3 核心流程
```
管理员登录 → 录入学生信息 → 注册人脸 → 选择课程 → 摄像头拍照识别 → 自动记录考勤 → 查看统计报表
```

### 1.4 产品目标
- 替代传统点名/签到方式，3 秒内完成一次人脸识别打卡
- 支持离线存储与批量操作
- 考勤数据可视化（出勤率、迟到/缺勤统计）
- 支持多班级、多课程管理

## 2. 界面总览

系统为 **桌面应用**（PyQt5），框架窗口，包含以下主界面：

### 2.1 页面结构
```
Login Page          ← 全屏居中登录卡片
└── Main Window     ← 登录后进入
    ├── 导航侧栏     
    ├── 页面堆栈     ← 右侧内容区，根据导航切换
    │   ├── 考勤总览  ← Dashboard / 首页
    │   ├── 人脸打卡  ← 摄像头识别
    │   ├── 考勤记录  ← 历史查询 + 统计
    │   ├── 学生管理  ← CRUD + 人脸注册
    │   └── 课程管理  ← CRUD
    └── 顶部标题栏   ← 窗口标题 + 最小化/关闭
```

## 3. 功能页面详细规格

### 3.1 登录页

**说明**：全屏遮罩，管理员输入用户名密码。


**交互逻辑**：
- Enter 键触发登录
- 登录按钮点击后置灰，显示"登录中..."
- 登录失败→显示红色错误文字（后端 message 字段）
- 登录成功→延迟 300ms 跳转到主窗口
- 10 分钟内密码输错超过 10 次会被限流


---

### 3.2 主窗口（Main Window）

**说明**：登录后的主界面容器。

---

### 3.3 考勤总览（Dashboard）

**说明**：首页看板，显示概览统计和快捷入口。

**数据来源**：
- 统计卡片：`GET /attendance/statistics`
- 快捷卡片：点击跳转对应导航页面（纯前端跳转）

**空态处理**：无数据时显示 "--"，不报错。

---

### 3.4 人脸打卡（Check-in）

**说明**：核心功能页，摄像头实时画面 + 拍照识别。


**交互逻辑**：
- 点击「启动摄像头」→ 打开摄像头，实时画面显示
- 点击「拍照识别」→ 截图 → 发送后端 `/face/recognize` → 右侧显示结果
- 识别成功：绿色 ✅ + 学生姓名 + 学号 + 时间 + 置信度
- 识别失败：黄色 ⚠️ 未识别 + 距离值
- 接口错误：红色 ❌ + 错误 message
- 可下拉选择课程（可选），选中后识别记录关联该课程

**摄像头状态**：
- 未启动：显示占位文字"点击「启动摄像头」开始"
- 已启动：实时画面流
- 无摄像头：提示用户连接摄像头

**空态处理**：
- 第一次进入：右侧结果区显示"等待识别..."
- 系统中无已注册人脸时：显示对应提示

---

### 3.5 考勤记录（Attendance Records）

**说明**：查询考勤明细 + 顶部统计摘要。


**筛选逻辑**：
- 日期范围：默认最近 7 天
- 课程：下拉从 `GET /courses` 获取
- 点击「查询」→ 带参数请求 `GET /attendance`
- 表格下方状态栏展示汇总统计（从 `/attendance/statistics` 获取）


**空态处理**：
- 无数据：表格显示一行 "暂无考勤记录"
- 筛选无结果：保留统计摘要归零

---

### 3.6 学生管理（Student Management）

**说明**：学生 CRUD + 人脸注册入口。


**交互逻辑**：
- 新增表单：填写学号、姓名、性别、下拉选择班级 → 点击「+ 添加」→ POST 请求 → 刷新表格
- 删除：点击「删除」→ 弹出确认对话框 "确定删除该学生？此操作不可撤销。" → 确认后 DELETE 请求
- 人脸注册：点击「📷 注册人脸」→ 弹出文件选择对话框（jpg/jpeg/png）→ 选择照片 → 自动上传注册
- 表格每行两个操作按钮，中间用间隔分开

**人脸状态列**：
- 已注册：绿色 "✅ 已注册" 
- 未注册：红色 "❌ 未注册"

**空态处理**：
- 无学生：表格为空，显示 "暂无学生，请添加"
- 人脸注册失败：弹窗显示错误 message

---

### 3.7 课程管理（Course Management）

**说明**：课程 CRUD。


**交互逻辑**：
- 表单三个输入后点击「+ 添加课程」→ POST 请求 → 清空输入框 → 刷新表格
- 删除：弹确认框 → DELETE 请求
- 如果课程有考勤记录，后端会返回 409，前端显示错误提示

**空态处理**：
- 无课程：表格一行 "暂无课程，请添加"

---

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 启动速度 | 从点击到登录页显示 < 3 秒 |
| 识别响应 | 拍照后 <= 5 秒返回结果 |
| 并发 | 单机使用，无需高并发 |
| 兼容性 | Windows 10/11 |
| 容错 | 后端不可用时显示友好提示，不崩溃 |
| 数据安全 | JWT 鉴权，口令 Argon2 加密 |
